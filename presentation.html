Problem Statement

Current Challenges

  • Ethiopian SACCOs face high loan default rates
  • Credit assessment is manual, slow & subjective
  • No standardized risk evaluation framework exists
  • Limited access to credit bureau data in Ethiopia
  • Inconsistent decisions across different officers

Impact

  • Financial losses threaten SACCO sustainability
  • Members' savings are put at risk
  • Loan processing delays frustrate borrowers
  • Human bias leads to unfair lending decisions
  • No data-driven insights for management

Objectives

General Objective

To design and develop an AI-powered loan default prediction system that assists Ethiopian SACCO microfinance institutions in making accurate, data-driven lending decisions.

Specific Objectives

  • Build a hybrid ML + expert-rules risk scoring engine
  • Engineer financial ratio features for accurate prediction
  • Implement critical fail safety rules for high-risk detection
  • Create an explainable AI system (not a black box)
  • Develop role-based dashboard (Officer, Committee, Admin)
  • Deploy lightweight model without Python runtime dependency
3 / 13
💡

Proposed Solution

🤖 ML Engine

  • Logistic Regression classifier
  • 14 features (10 raw + 4 engineered ratios)
  • Trained on 1,500 synthetic borrower profiles
  • Exported as JSON — runs in JavaScript

🛡️ Expert Rules

  • 6 weighted risk factors (domain expert logic)
  • 4 critical fail hard-override conditions
  • Enforced both in training and at runtime
  • Safety net against ML misclassification

📊 Explainable Output

  • Risk score 0–100% with color coding
  • Human-readable ✅⚠️❌ explanations
  • Feature breakdown for each prediction
  • Recommendation: Approve / Modify / Reject

Critical Fail Conditions

Hard rules that override ML score to 95% risk

Rule 1: Payment > Income

Monthly payment exceeds disposable income — borrower literally cannot pay

Rule 2: DTI > 50%

Debt-to-income ratio exceeds international banking threshold

Rule 3: Loan > 3× Savings

Loan amount exceeds 3 times total savings — extreme overextension

Rule 4: No Collateral

Collateral value is zero — no recovery option for the SACCO

Challenges Faced

Technical Challenges

  • No real SACCO loan dataset available — solved with synthetic data generation
  • Class imbalance in training data — solved with balanced class weights
  • Model gave incorrect low-risk to high-risk applicants — solved with critical fail rules
  • Feature correlations were misaligned — solved with expert-weighted scoring

Design Challenges

  • Making AI predictions explainable to non-technical users
  • Ensuring model runs without Python server at runtime
  • Balancing ML flexibility with rigid financial rules
  • Designing role-based access for different decision levels
